Mace Azure Setup Guide

MaceStyle: No-AI Production

The Mace-side steps to stand up the Function and let me deploy it

For the Mace Azure / Digital team, from Stephen Cummins · AI Service Lead, Mace Digital 8 July 2026
MaceStyle can go to production this sprint, running entirely inside Mace's Azure and Microsoft 365. In this mode the AI rules are switched off, so no document text leaves Mace — no external data, no governance wait. It enforces the deterministic core of the Style Guide (British spelling, contractions, symbols, numbers, fonts, colours). AI-powered judgement rules layer on later with zero rework — same codebase, one setting. This sheet covers the three things Mace needs to do so I can deploy and finish the wiring.

Prerequisites

  • A Mace Azure subscription + resource group (UK South) to own the service.
  • Someone with Owner/Contributor on that resource group to run Steps 2–3, and access to the app registration (Step 1).
  • No AI resource, no Marketplace step, no US data zone — this mode needs none of it.

Step 1 — Reuse the existing app registration Mace

MaceStyle authenticates to SharePoint with an app registration that already exists on the Mace tenant and already has the right permission granted. Do not create a new one — just confirm and supply a secret:

Send the secret to me via a secure channel (or, preferred, drop it straight into the Function App settings / Key Vault in Step 2 so I never handle the raw value).

Step 2 — Create the Function App Mace

Create an Azure Function App in the Mace resource group with these settings:

TypeFunction App — Flex Consumption plan
RuntimePython 3.11, Linux
RegionUK South
Namee.g. func-macestyle-prod (Mace convention)

A storage account is created automatically. Nothing else is needed at creation time — I set the app configuration as part of the deploy.

Step 3 — Give me deploy access Mace

Grant my Mace account (stephen.cummins@macegroup.com) an RBAC role on the Function App (or the resource group) so I can deploy the code and set its configuration from VS Code:

RoleContributor on the Function App (or Website Contributor if you prefer least-privilege)
ScopeThe Function App from Step 2 (or its resource group)
Assigneestephen.cummins@macegroup.com

Hand me these three things

  • Subscription ID + resource group name + the Function App name/region (Step 2)
  • Confirmation my account has the deploy role (Step 3)
  • The app-registration client secret — securely, or pre-loaded into app settings (Step 1)

What I do next Me

Once the above is in place, no further Mace admin effort is needed. I will:

  1. Deploy the code to the Function App from VS Code.
  2. Set the app settings — the SharePoint credentials + list IDs, plus ENABLE_CLAUDE_AI=false (the switch that keeps it AI-free) and the endpoint access key. No AI variables are set in this mode.
  3. Re-point the existing Power Automate flow ("Mace Document Style Validator") at the new Function URL.
  4. Verify end-to-end on a test document and confirm the deterministic corrections apply.
